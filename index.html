BlockCoin Documentation

Welcome to the official documentation for the BlockCoin Ledger ecosystem. This system orchestrates and records balances across multiple distributed gaming instances on the Scratch network securely using a specialized back-end queue processor.

Project Setup & Integration

To implement BlockCoin into your own custom Scratch games, follow these implementation steps:

1. Backpack the Request Engine
Open the official master project editor at https://scratch.mit.edu/projects/1331635541/editor/, open your backpack menu, and drag the sprite named "Cloud Requests" inside. Paste this sprite directly into your target game project.
2. Mandatory Project Credits
This network relies on the cloud request infrastructure created by @timmccool. You MUST provide explicit credit to @timmccool within your project description fields before sharing your game online.

BlockCode Manager (Official Layouts)

Below are the accurate blocks required to execute secure payment transactions within your game interface.

Official Payment Request Function (PayCoins)

define PayCoins (Amount)
Send Request | request name: [get_captcha] argument1: (username) and wait :: custom
ask [response] and wait
Send request | request name: [get_owner] and wait :: custom
ask (join [Do you authorize a transaction of ] (join (Amount) (join [ to ] (join (response) [ (y/n)])))) and wait
if <(answer) = [y]> then
  Send request | request name: [transaction] argument1: (Amount) argument2: (username) and wait :: custom
end

Verification Script Parsing Context

when this sprite clicked
ask [Enter your Scratch username:] and wait
set [username v] to (answer)
set [server_response v] to (brackets [get_captcha] (username) :: extension)
if <(server_response) contains [|]> then
  set [idx v] to [1]
  set [rule_text v] to []
  repeat until <(letter (idx) of (server_response)) = [|]>
    set [rule_text v] to (join (rule_text) (letter (idx) of (server_response)))
    change [idx v] by (1)
  end
  change [idx v] by (1)
  set [verification_code v] to []
  repeat until <(idx) > (length of (server_response))>
    set [verification_code v] to (join (verification_code) (letter (idx) of (server_response)))
    change [idx v] by (1)
  end
  say (rule_text) for (5) secs
  say (join [Your code is: ] (verification_code))
else
  say [Server error. Please try again.] for (3) secs
end

Environment Setup

The backend requires specific secure variables exposed inside the hosting instance environment to link up with your Scratch account cleanly.

# Required System Context Keys
export SCRATCH_USER2="YourScratchAccountUsername"
export SCRATCH_PASS2="YourSecureProfilePassword"

Central Queue Engine

To safely handle data modifications without letting Scratch drop incoming variables due to high traffic volume, all listening wrapper contexts defer execution tasks to a synchronized, thread-safe queue.

# Internal execution delay buffer to secure connection safety
time.sleep(0.15)
