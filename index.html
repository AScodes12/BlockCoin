Welcome to the official documentation for the BlockCoin Ledger ecosystem. This system orchestrates and records balances across multiple distributed gaming instances on the Scratch network securely using a specialized back-end queue processor.

BlockCode Manager (Scratch Layouts)

Below are the accurate layouts required to build the interface inside your Scratch projects. These are rendered live using standard forum syntax text syntax.

1. Requesting the Verification Code

when this sprite clicked
ask [Enter your Scratch username:] and wait
set [username v] to (answer)
set [server_response v] to (brackets [get_captcha] (username) :: extension)
if <(server_response) contains [|]> then
  set [idx v] to [1]
  set [rule_text v] to []
  repeat until <(letter (idx) of (server_response)) = [|]>
    set [rule_text v] to (join (rule_text) (letter (idx) of (server_response)))
    change [idx v] by (1)
  end
  change [idx v] by (1)
  set [verification_code v] to []
  repeat until <(idx) > (length of (server_response))>
    set [verification_code v] to (join (verification_code) (letter (idx) of (server_response)))
    change [idx v] by (1)
  end
  say (rule_text) for (5) secs
  say (join [Your code is: ] (verification_code))
else
  say [Server error. Please try again.] for (3) secs
end

2. Checking Comment Status

when this sprite clicked
if <(verification_code) = []> then
  say [Please request a code first!] for (3) secs
  stop [this script v]
end
say [Checking project comments...] for (2) secs
set [verify_status v] to (brackets [send_captcha] (username) (verification_code) :: extension)
if <(verify_status) = [VALIDATED]> then
  say [Success! Proceeding to transaction...] for (3) secs
else
  if <(verify_status) = [ERROR_NOT_COMMENTED_YET]> then
    say [Code not found yet. Make sure you commented on the project!] for (4) secs
  else
    say (join [Verification failed: ] (verify_status)) for (3) secs
  end
end

Environment Setup

The backend requires specific secure variables exposed inside the hosting instance environment to link up with your Scratch account cleanly.

# Required System Context Keys
export SCRATCH_USER2="YourScratchAccountUsername"
export SCRATCH_PASS2="YourSecureProfilePassword"

Central Queue Engine

To safely handle data modifications without letting Scratch drop incoming variables due to high traffic volume, all listening wrapper contexts defer execution tasks to a synchronized, thread-safe queue. The background worker evaluates inputs sequentially, injecting a constant safe delay interval:

# Internal execution delay buffer to secure connection safety
time.sleep(0.15)

Comment Verification System

The authorization layer uses native project-comment confirmation protocols rather than standard math puzzles to prevent automation and script exploits.

Critical System Requirement
The MASTER_PROJECT_ID defined at the top of your server configuration file must point to an active, publicly shared Scratch project with public comments enabled. If the project remains private or unshared, lookups will fail with a ProjectNotFound error.
