BlockCoin Documentation

Welcome to the official documentation for the BlockCoin currency system for Scratch[cite: 2]. This system uses scratchattach to connect shops and games to a centralized server and uses a captcha to verify transactions safely and easily[cite: 3].

How to Add BlockCoin to Your Project

  1. Visit the official BlockCoin project page at scratch.mit.edu/projects/1331635541/[cite: 5].
  2. Open your Backpack panel at the bottom of the Scratch editor[cite: 5].
  3. Backpack the "Cloud Requests" sprite and add it into your project[cite: 5].
  4. Required Credit: You must explicitly credit @Timmccool in your project description or notes and credits for providing the cloud framework[cite: 6].

How to Get Approval to Use BlockCoin

To ensure network stability, all project links must be approved before the server will run requests.

  1. Visit the official BlockCoin forum thread (link) and fill out the system application[cite: 8].
  2. Approval usually takes between 24 to 48 hours[cite: 8].
  3. In the meantime, the centralized Python server will not process any requests coming from your project ID[cite: 9].
🛠️ Advanced Topics

Optimized Transaction Timeline

The transaction workflow processes item distribution context entirely on the backend server. When a client checks out an item, run this simplified block sequence:

  1. Call transaction passing [Amount:Item] as Argument 1, and username as Argument 2.
  2. Verify the engine returns a SUCCESS token back to your local client script.
  3. Instantly dispatch a call to get_item passing username as Argument 1. The server handles verification context automatically.

BlockCoin Commands

Command Description
get_captcha Returns a comment verification captcha code to your project canvas[cite: 17].
captcha_sent Tells the Python server to check the master project comments for the user's validation code.
get_owner Finds the registered owner of the project on the server (all transactions will be paid to the owner)[cite: 18].
transaction Pays an amount to the project owner by verifying balances and creating a pending transaction token[cite: 19].
get_item Claims the pending transaction receipt and saves the item permanently to the user's database portfolio.
get_inventory Returns a comma-separated list of all items owned by the user inside the calling project's shop.

Safety Features

BlockCoin contains built-in safety features to preserve the integrity of projects on the server and the users using them[cite: 14]:

  • Platform Verification: Uses Scratchattach's built-in verification system to confirm identity logs[cite: 15].
  • Spam Filters: Uses anti-bot scripts and request limits on the server side to stop malicious traffic[cite: 15].
  • Single Ownership Limits: The database enforces that specific items can only be bought once per user per project to keep profile inventories consistent.

Quick Troubleshooting

Problem Solution
Transactions are stuck loading Make sure your project has gone through the forum application and your project ID is whitelisted on the server database.
Verification codes are failing Remind users they must comment the exact captcha puzzle string directly on the official master project profile[cite: 12].
ERROR_NO_PENDING_TRANSACTION Your project is calling get_item without receiving a SUCCESS message from the transaction command first. Check that the user has enough money and does not already own the item.
Cloud variables disconnected Platforms experience occasional load drops during peak traffic hours. Users can safely restart the cloud loop by clicking the green flag.
