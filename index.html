BlockCoin Documentation

Welcome to the official documentation for the BlockCoin currency system for Scratch. This system uses Scratchattach to connect shops and games to a centralized server and then uses a captcha to verify transactions safely and easily.

How to Add BlockCoin to Your Project

  1. Visit the official BlockCoin project at scratch.mit.edu/projects/1331635541/.
  2. Open your Backpack panel at the bottom of the Scratch editor.
  3. Drag the "Cloud Requests" sprite down into your backpack, open your target project, and drop the sprite inside.
  4. Required Credit: You must explicitly credit @Timmccool in your project description or notes and credits section for providing the underlying cloud framework.

How to Get Approval to Use BlockCoin

To keep the network secure, all project links must be whitelisted before processing live requests.

  1. Visit the official BlockCoin forum thread (link) and fill out the application.
  2. Approval timelines typically take between 24 to 48 hours.
NOTICE: In the meantime, the central server will automatically ignore and fail to process any network requests originating from your unapproved project ID.

How to Use BlockCoin & Secure Asset Creation

To execute financial exchanges or payments using BlockCoin, implement the custom script block PayCoins (amount).

This block communicates with our Python backend to request a unique transaction code. The player is required to type this verification code as a comment on the official root project page to authorize the payment.

IMPORTANT FOR SHOP DEVELOPERS: 2-ARGUMENT CLOUD WORKFLOW
Due to strict platform limitations, requests accept a maximum of 2 arguments. When a player purchases an item, follow this exact sequence to commit the item transaction safely into python server storage:
  1. Call transaction passing [cost:item_name] as Arg 1, and username as Arg 2.
  2. Upon success, dispatch a request to get_project_id to fetch the active runtime context string.
  3. Instantly dispatch a call to get_item passing username as Arg 1, and the combined [project_id:item_name] string as Arg 2.

BlockCoin Commands API

These are the underlying API endpoint triggers handled by the custom code logic inside the imported Cloud Requests sprite module:

Command Name What It Does
get_captcha Requests a unique comment verification captcha code and returns it to your game project client.
captcha_sent Informs the python backend to sweep the master project for the completed captcha validation comment token.
get_owner Finds the registered Scratch username of the project creator on the central server (all transaction currency transfers automatically deposit to this account).
transaction Processes currency extraction. Accepts a complex string parameter parsed as cost:item_name to flag downstream item claims.
get_project_id Returns the server-side runtime project target ID to provide security auditing contexts back to the client player.
get_item Validates the active transaction token. Receives a structured project_id:item_name chunk to claim items securely into permanent server storage.
get_inventory Returns a unified, comma-separated string containing every validated item owned by the user profile for easy list parsing.

Safety & Platform Integrity

BlockCoin includes standard structural parameters on the network layers to ensure balanced environment behaviors across games:

  • Identity Checks: Uses Scratchattach’s built-in platform validation methods to establish account identity.
  • Anti-Replay Ledger Tracking: Clears transaction receipts immediately following a get_item claim sequence so assets cannot be duplicated or intercepted.
  • Network Security: Enforces automated anti-bot scripts and request spam limits directly on the central server instance to guard against data flooding.

Quick Troubleshooting

Issue Encountered Resolution Path
Transactions are completely stuck on loading Ensure that your project has gone through the forum application process and has been verified by an administrator. Unverified projects cannot connect.
Verification codes are failing Remind your players that they must submit the accurate text token as a direct comment on the official project page to clear the captcha wall.
Items are failing to save to python database Ensure your client code executes get_item within 60 seconds of a transaction before the verification token securely expires in server memory.
Cloud variable connection dropped Scratch cloud limits can occasionally skip variables during peak times. The backend queue handles this, but players can safely restart the game script if an input fails to register.
